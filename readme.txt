## 1. Работа с библиотекой requests, http запросы
### Дополнительно хочется раскрыть темы:
- Что такое HTTP;
- Заголовки;
- Методы HTTP, соглашения;
- GET, POST. Их отличие, применение;
- другие: PUT, DELETE;
- виды передачи данных: get-параметр, json (файлы, данные в заголовке);
- статусы в response;
- Кейсы применения: фронтенд-бекенд, взаимодействие между сервисами;
- Библиотека requests, ссылка доки;

### Знает:
- общее представление о протоколе, область его применения;
- общее представление о заголовках;
- представление о возвращаемых статусах request’а;
- знание основных методов GET, POST (общее понимание PUT, DELETE);
- представление о передаваемой информации;
- понимать когда какой метод применять;

### Умеет:
- умеет определять методы с помощью requests;
- умеет формировать request с get-параметрами, json параметрами, файлы, новые заголовки;
- делать get-запрос с параметрами;
- делать post-запрос с параметрами;
- обрабатывать response запроса: json, получение статуса;

### Практика:

#### В классе:
1) Конструирование запросов руками: браузер, postman, network (chrome);
2) Сначала покажу результат запросов на http://requestb.in/.
3) Сделаем пару запросов на get с http://httpbin.org/, выведем данные.
4) Сделаем карманный переводчик:
    - подключимся к сервису по переводу, получим токен;
    - напишем функцию, которая принимает русский текст, а возвращает перевод на английском;

#### Домашка:
 Расширить функцию, чтобы она принимала параметры: путь к файлу с текстом; путь файла с результатом;
 язык с которого перевести; язык на который перевести=RUS.
 
 Дадим 3 файла с новостями на 3 языках: французский, испанский, немецкий.

 Функция должна взять файл с текстом, перевести его и сохранить результат в другой файл.
 

#### Критерии проверки ДЗ
1) Должна использоваться библиотека requests;
2) Функция должна успешно прочитать, перевести и сохранить;
